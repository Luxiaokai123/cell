
# 细胞智能计数系统 – 软件开发需求清单 v1.0

## 1. 总体目标
开发一款基于 Web 的原型验证系统，集成 RDHS‑YOLO（实例分割）与 DAS‑DETR（目标检测）两种细胞计数算法，并具备智能模型选择能力。系统通过可视化界面展示细胞检测/分割结果及统计数据，验证算法的有效性与实用性。

## 2. 用户角色
- **普通用户**：上传图像、查看结果、导出数据。
- **管理员/研究者**：可手动切换模型、查看模型决策依据（如特征值）。

## 3. 功能需求

### 3.1 图像输入模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 单张上传 | 支持 JPEG/PNG 格式，单个文件 ≤ 20MB | P0 |
| 批量上传 | 支持上传文件夹（最多 50 张），后台排队处理 | P1 |
| 示例图像 | 内置 3~5 张典型细胞图像供快速体验 | P0 |
| 图像预览 | 上传后显示缩略图 | P0 |

### 3.2 智能模型选择模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 自动模式（默认） | 系统根据图像内容自动选择最优模型 | P0 |
| 手动模式 | 用户强制指定使用 RDHS‑YOLO 或 DAS‑DETR | P0 |
| 决策依据展示 | 显示自动选择的理由（如密度、平均大小等统计值） | P1 |
| 预分析模块 | 调用轻量级检测器（YOLOv8n）快速提取图像特征 | P0 |

**自动选择规则（示例）**：
- 预检测得到细胞数量 N，平均框面积 A。
- 若 N > 50 且 A < 100 像素 → 选用 DAS‑DETR（密集微小场景）
- 否则 → 选用 RDHS‑YOLO（一般场景或需精细轮廓）

### 3.3 模型推理模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 后端模型加载 | 加载 RDHS‑YOLO 和 DAS‑DETR 的权重文件（.pth） | P0 |
| 推理执行 | 接收前端图像，调用对应模型推理 | P0 |
| 超时处理 | 单张推理超过 30 秒则返回超时提示 | P1 |
| 批量推理 | 对批量图像自动串行处理，显示进度 | P1 |

### 3.4 结果可视化模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 检测框绘制 | DAS‑DETR 输出边界框、类别标签、置信度 | P0 |
| 分割掩码绘制 | RDHS‑YOLO 输出不同颜色的实例轮廓，透明填充 | P0 |
| 细胞总数显示 | 界面右上角或底部显示当前图像检测到的细胞数量 | P0 |
| 置信度过滤 | 提供滑动条（0~1）动态调整显示阈值 | P1 |
| 叠加显示 | 可勾选是否显示框/掩码，方便与原图对比 | P1 |
| 特征图可视化 | 可选展示 DCSA 注意力热图或 Agent 注意力图（用于论文展示） | P2 |

### 3.5 结果导出模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 保存结果图 | 将带标注的图像保存为 PNG | P0 |
| 导出统计表 | 导出 CSV 文件，包含：文件名、细胞数量、平均面积、类别分布等 | P1 |
| 批量导出 | 批量处理完成后打包下载所有结果图及汇总表格 | P1 |

### 3.6 历史记录模块（可选）
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 会话记录 | 保存当前会话的上传记录与结果（刷新后清空） | P2 |
| 日志 | 记录推理时间、所选模型等信息（供调试） | P2 |
